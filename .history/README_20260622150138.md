# Instagram Engagement Scraper

## Project Purpose
This repository contains a Python scraper for collecting **Instagram content engagement data**.
The user provides a list of Instagram post/reel URLs, and the scraper extracts key engagement
metrics for each content item, then uploads the results to a Google Sheets worksheet as a
structured engagement report.

## Target Data Per Content
For each Instagram URL provided, the scraper collects:
- **Likes** — total number of likes on the post/reel
- **Comments** — total number of comments
- **Reposts** — number of times the content was reposted (where available)
- **Saves** — number of saves (where available)
- **Shares** — number of shares (where available)
- **Views** — total view count (for Reels and videos)
- **Content Type** — Post / Reel / Carousel
- **Caption** (optional) — first N characters of the caption
- **Post Date** — publication timestamp
- **URL** — the original link provided
- **Ingestion Timestamp** — when the data was collected

## Current Status
- The main entry point is `main.py`.
- URL input is read from a local text file (`input/urls.txt`) or passed directly via CLI.
- Scraping is handled by `scraper/instagram_scraper.py` using requests, BeautifulSoup,
  and/or Playwright for JavaScript-rendered content.
- Data is normalized and validated in `transformer/`.
- Google Sheets upload is performed by `services/spreadsheet_service.py`.
- The service writes all rows to a single worksheet tab named `Engagement Report` by default.
- Each row also includes an `Ingestion.Timestamp` value.
- Logging is written to `logs/scraper.log`.
- Environment configuration is loaded from `.env` via `config/settings.py`.

## Important Notes
- Instagram content is JavaScript-rendered; the scraper uses Playwright or a headless browser
  to fully load the page before extracting data.
- Some metrics (saves, reposts, shares) may not be publicly visible on all accounts; the
  scraper will record `N/A` for unavailable fields.
- Rate limiting: add a delay between requests to avoid being blocked by Instagram.
- The `.env` file must define `GOOGLE_SHEET_ID`, and either `GOOGLE_CREDENTIAL_PATH`
  or `GOOGLE_CREDENTIAL_PATH_B64`.

## Input Format (`input/urls.txt`)

```

```

## How to Run
1. Activate the Python virtual environment.
2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Confirm `.env` contains correct values:
   ```
   GOOGLE_SHEET_ID=your_sheet_id_or_url
   GOOGLE_CREDENTIAL_PATH=credentials/your_service_account.json
   GOOGLE_WORKSHEET_NAME=Engagement Report
   TIMEZONE=Asia/Jakarta
   ```
4. Add Instagram URLs to `input/urls.txt` (one per line).
5. Run:
   ```bash
   python main.py
   ```

## Scraping Strategy Options
The scraper can use one of two approaches (configurable in `.env` via `SCRAPER_STRATEGY`):

1. **Playwright (default)** — launches a headless browser, loads the Instagram page fully,
   then parses the HTML for engagement numbers. More reliable but slower.
2. **Instaloader** — uses the `instaloader` Python library which calls Instagram's internal
   API. Faster but may require Instagram login credentials.

Set in `.env`:
```
SCRAPER_STRATEGY=playwright   # or: instaloader
INSTAGRAM_USERNAME=           # required if SCRAPER_STRATEGY=instaloader
INSTAGRAM_PASSWORD=           # required if SCRAPER_STRATEGY=instaloader
```

## Logs
- Runtime logs are written to `logs/scraper.log`.
- Check this file first when debugging scraping or upload failures.

## Known Limitations
- Instagram may block scraping attempts; use delays and rotate user agents.
- Private accounts will return no data.
- Metrics like saves and shares are not always publicly visible.
- Instagram's HTML structure changes frequently; the scraper selectors may need updates.
