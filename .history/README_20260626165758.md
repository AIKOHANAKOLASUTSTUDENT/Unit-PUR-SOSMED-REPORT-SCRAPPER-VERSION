# Instagram Engagement Scraper (Streamlit + Sheets)

## Project Purpose

This repository provides a Streamlit-based tool and scraper to collect Instagram post/reel
engagement data and export a structured report (CSV or Google Sheets). You paste or load
Instagram URLs, run the scraper, review results in the dashboard, then confirm to copy
the results to Google Sheets.

## Target Data Per Content

For each Instagram URL the tool attempts to produce:

- Likes тАФ total likes
- Comments тАФ total comments
- Views тАФ view count (for Reels/videos)
- Reposts / Saves / Shares тАФ when available, otherwise `N/A`
- Content Type тАФ `Post`, `Reel`, or `Carousel`
- Caption preview тАФ first N characters
- Post Date тАФ publication timestamp (normalized to configured timezone)
- URL тАФ original link
- Ingestion Timestamp тАФ when the record was collected
- Collab Status тАФ one of `konten akun cbp`, `sudah collab`, or `belum collab`
- Reach (display) тАФ numeric reach for content uploaded by the CBP account; a textual
  label `bukan konten dari akun cbp KPwBI Sulut` for other accounts

## Current Status

- Frontend: `dashboard.py` (Streamlit) тАФ paste or upload URLs, run scraper, review results.
- Scraper: `scraper/instagram_scraper.py` тАФ routes `/reel/` and `/p/` URLs through
  Instaloader (preferred) with fallbacks to Meta API and Apify when needed.
- Transformers: `transformer/` тАФ normalizer, processor (adds `reach_display` and `collab_status`),
  and validator.
- Sheets: `services/spreadsheet_service.py` тАФ appends rows to a single worksheet `Engagement Report`.
- Upload behavior: the dashboard does NOT automatically write to Sheets тАФ a confirmation
  button `тЬЕ Konfirmasi Salin ke Google Sheets` must be pressed to perform the append.
- Sorting: results are shown and appended to Sheets oldest тЖТ newest (ascending by post date).
- Logging: `utils/logger.py` writes runtime logs to console and files under `logs/`.

## Important Notes

- The scraper prefers `Instaloader` for post/reel metadata. When Instaloader cannot fetch
  a post, the code falls back to Meta Graph API (if configured) and then Apify.
- `playwright` and `beautifulsoup4` are used as optional fallbacks for JS-rendered pages;
  install them (and run `playwright install`) to enable all fallback modes.
- Reach handling: if the post uploader matches `INSTAGRAM_USERNAME` (CBP account), the
  dashboard shows numeric reach (from Meta/insights when available). For other uploaders,
  the Reach column displays the text `bukan konten dari akun cbp KPwBI Sulut`.
- Collab detection: `normalize_collab_status()` returns:
  - `konten akun cbp` if uploader equals `INSTAGRAM_USERNAME`
  - `sudah collab` if the caption mentions CBP handle
  - `belum collab` otherwise
- Google Sheets errors (e.g. API 500/503) are logged and shown in the UI; they do not
  crash the dashboard. Use the confirm button to retry after fixing credentials or sheet access.

## Input Format (`input/urls.txt`)

One Instagram post or reel URL per line. Example:

```
https://www.instagram.com/p/ABC123/
https://www.instagram.com/reel/XYZ789/
https://www.instagram.com/p/DEF456/
```

## Output Format (Google Sheets)

Each row in the sheet represents one content item:

| URL         | Content Type | Post Date  | Likes | Comments | Views  | Reposts | Saves | Shares | Caption (preview)   | Ingestion Timestamp |
| ----------- | ------------ | ---------- | ----- | -------- | ------ | ------- | ----- | ------ | ------------------- | ------------------- |
| https://... | Reel         | 2024-06-01 | 12400 | 340      | 980000 | N/A     | N/A   | N/A    | "Check out this..." | 2024-06-22 10:00:00 |

## How to Run (recommended)

1. Create and activate a virtual environment (Windows example):

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1   # PowerShell
# or: .\.venv\Scripts\activate.bat  # cmd.exe
```

2. Install Python dependencies:

```powershell
pip install -r requirements.txt
python -m playwright install
```

3. Create a `.env` file with required settings (example):

```text
GOOGLE_SHEET_ID=your_sheet_id_or_url
GOOGLE_CREDENTIAL_PATH=credentials/service_account.json
GOOGLE_WORKSHEET_NAME=Engagement Report
TIMEZONE=Asia/Jakarta
INSTAGRAM_USERNAME=cbp.rupiah_qris_peka_bi_sulut
META_ACCESS_TOKEN=<optional_meta_token>
META_INSTAGRAM_ID=<optional_instagram_id>
SCRAPER_STRATEGY=hybrid
```

4. Run the Streamlit dashboard from the project root (ensure venv is active):

```powershell
.\.venv\Scripts\python.exe -m streamlit run dashboard.py
```

5. In the dashboard: paste Instagram URLs, click `Mulai Scraping`, review results, then
   press `тЬЕ Konfirmasi Salin ke Google Sheets` to append rows to the sheet.

## Dependencies

Key dependencies are listed in `requirements.txt`. Notable ones:

- `playwright` тАФ browser-based scraping for JS content (run `python -m playwright install`).
- `beautifulsoup4` тАФ HTML parsing fallback.
- `instaloader` тАФ optional faster metadata fetch for public posts.
- `gspread`, `google-auth` тАФ Google Sheets integration.

## Scraping Strategy

Set `SCRAPER_STRATEGY` in `.env` to control preferred scraping method (`instaloader`,
`meta`, `apify`, or `hybrid`). The scraper prefers `instaloader` for `/reel/` and `/p/` URLs
and falls back to Meta Graph API or Apify when necessary.

If you use `playwright` for fallback HTML scraping, ensure you installed Playwright browsers
(`python -m playwright install`).

## Logs

- Runtime logs are written under `logs/`.
- If Google Sheets returns API errors (500/503), the dashboard shows the message and keeps
  the scraped data in the UI so you can retry after fixing credentials or permissions.

## Known Limitations

- Instagram may block or throttle scraping. Use appropriate delays and authenticated
  requests when possible.
- Private accounts return limited/no public data.
- Some metrics are not publicly exposed (saves, shares) and will display `N/A`.
